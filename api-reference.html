ShopFlow API (1.0.0)

Download OpenAPI specification:Download

ShopFlow API Support: support@shopflow.com URL: https://shopflow.com/support

Overview

The ShopFlow API is a RESTful API that allows you to build online stores, manage products, handle orders, and process payments programmatically.

 

 

Getting Started

In this guide you will make your first ShopFlow API call and retrieve your product catalogue. This takes about 5 minutes.

 

Step 1 — Prerequisites

Before you begin, make sure you have the following:

Requirement Details
ShopFlow account Required to generate your API key
Node.js 18+ Required to run the code sample
npm or yarn Comes bundled with Node.js
Terminal Terminal on Mac, Command Prompt on Windows

Install the ShopFlow SDK:

npm install shopflow-sdk

 

Step 2 — Get your API key

  1. Sign in to the ShopFlow platform.
  2. Navigate to Accounts → Settings → API Keys.
  3. Click Create New Key.
  4. Copy and save the generated API key.

Keep your API key secret. Never commit it to GitHub or share it publicly. Use environment variables in production.

 

Step 3 — Make your first API call

The following example retrieves all products from your ShopFlow catalogue:

// Install the SDK: npm install shopflow-sdk
const ShopFlow = require('shopflow-sdk');

// Initialise with your API key
const client = new ShopFlow({
  apiKey: 'YOUR_API_KEY' // Replace with your actual key
});

// Make your first API call — retrieve all products
async function getProducts() {
  const products = await client.products.list();
  console.log(products);
}

getProducts();

 

Step 4 — View the response

A successful request returns a JSON object containing your product catalogue:

{
  "products": [
    {
      "id": "prod_001",
      "name": "Sony WH-1000XM5 Headphones",
      "price": 29999,
      "stock": 45
    }
  ],
  "total": 1
}

You authenticated with the ShopFlow API using your API key and retrieved your product catalogue. The response is a JSON object containing an array of products and a total count.

 

Next steps

Now that you have made your first API call:

  • Add a new product — POST /products
  • Update an existing product — PATCH /products/{id}
  • Create an order — POST /orders
  • Explore the full reference — see the endpoints below

 

 

Authentication

ShopFlow uses API key authentication. Include your API key in the X-API-Key header with every request.

curl -H "X-API-Key: YOUR_API_KEY" https://api.shopflow.com/v1/products

Why API keys? ShopFlow uses API keys because they are simple, stateless, and ideal for server-to-server communication. For user-facing applications where your users need their own access, consider using OAuth 2.0 instead.

 

 

Rate Limits

Plan Requests per minute Requests per day
Free 60 1,000
Pro 300 50,000
Enterprise Unlimited Unlimited

When you exceed the rate limit, the API returns a 429 Too Many Requests response. Use the Retry-After header value to know when to retry.

 

 

Base URL

https://api.shopflow.com/v1

All endpoints are relative to this base URL.

Products

Manage your product catalogue. Use these endpoints to create, retrieve, update, and delete products in your ShopFlow store.

List all products

Returns a paginated list of all products in your ShopFlow catalogue. Use the category query parameter to filter by product type.

Authorizations:
ApiKeyAuth
query Parameters
category
string
Enum: "electronics" "clothing" "books" "home" "sports"
Example: category=electronics

Filter products by category.

limit
integer [ 1 .. 100 ]
Default: 20
Example: limit=20

Maximum number of products to return. Defaults to 20.

offset
integer
Default: 0
Example: offset=0

Number of products to skip for pagination. Defaults to 0.

Responses

Response Schema: application/json
Array of objects (Product)
total
integer

Total number of products matching the query.

Errors

Error Reference

ShopFlow uses standard HTTP status codes to indicate the success or failure of an API request.

HTTP Status Codes

Code Status When it occurs
200 OK Request succeeded
201 Created Resource created successfully
400 Bad Request Invalid parameters or missing required fields
401 Unauthorized Missing or invalid API key
403 Forbidden Valid API key but insufficient permissions
404 Not Found Resource does not exist
409 Conflict Resource already exists
422 Unprocessable Entity Validation error on request body
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Unexpected server error

Error Response Format

All errors follow this consistent format:

{
  "error": {
    "code": "PRODUCT_NOT_FOUND",
    "message": "The product with ID 'prod_999' does not exist.",
    "suggestion": "Use GET /products to retrieve valid product IDs.",
    "docs": "https://api.shopflow.com/errors/PRODUCT_NOT_FOUND"
  }
}

Error Code Reference

Error Code Status Cause Fix
INVALID_API_KEY 401 API key is malformed or missing Check your API key format and header name
EXPIRED_API_KEY 401 API key has been revoked Generate a new key in Settings → API Keys
INSUFFICIENT_PERMISSIONS 403 API key lacks required scope Use a key with read/write permissions
PRODUCT_NOT_FOUND 404 Product ID does not exist Use GET /products to get valid IDs
ORDER_NOT_FOUND 404 Order ID does not exist Use GET /orders to get valid IDs
DUPLICATE_PRODUCT 409 Product with same SKU exists Use a unique SKU or update existing product
VALIDATION_ERROR 422 Required field missing or invalid Check the error details for the specific field
RATE_LIMIT_EXCEEDED 429 Too many requests Wait for Retry-After duration before retrying
INTERNAL_ERROR 500 Unexpected server error Retry after a few seconds; contact support if it persists

Handling Errors in Code

async function getProduct(id) {
  try {
    const product = await client.products.get(id);
    console.log(product);
  } catch (error) {
    if (error.status === 404) {
      console.error('Product not found:', error.code);
      // Handle gracefully — show user a "not found" message
    } else if (error.status === 429) {
      const retryAfter = error.headers['Retry-After'];
      console.error(`Rate limited. Retry after ${retryAfter} seconds.`);
    } else {
      console.error('Unexpected error:', error.message);
    }
  }
}
