We're excited to announce ShopFlow API v2.0. This release adds two new endpoints, performance improvements, expanded SDK support, and breaking changes to address in your integration.

Two new endpoints are available. `GET /products/featured` lets you retrieve a curated list of featured products. `POST /orders/bulk` lets you create more than one order in a single request, reducing the number of API calls needed for high-volume workflows.

v2.0 includes three enhancements:

- **Faster response times.** Latency is reduced by 40% across all endpoints.
- **Better error messages.** All error responses now include a `suggestion` field to help you resolve issues faster.
- **New SDK support.** Official SDKs for Swift and Kotlin are now available.

v2.0 includes two breaking changes. The `price` field now returns values in rupees instead of paise, so update any display or calculation logic that assumed paise. `GET /products` now returns 20 results per page by default instead of 10, so update any pagination logic that relies on the previous default.

To learn more, see [What's new in ShopFlow API v2.0](/docs/whats-new).
